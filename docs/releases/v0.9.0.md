## Database Isolation

**Critical bug found:** Tests were writing to production database.

- **Root cause:** ES module import hoisting — `tests/setup.ts` couldn't override `DATABASE_URL` before PrismaClient instantiated
- **Fix:** `vitest.config.ts` injects `DATABASE_URL` as `define`; `db.ts` adds safety-net that aborts on wrong database
- **Verification:** Dedicated isolation test proves the mechanism
- **Permanent protection:** Double guard (config-time + runtime)
